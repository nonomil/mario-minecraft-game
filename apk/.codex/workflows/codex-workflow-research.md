# codex-workflow-research｜研究调研流程（Codex 主导）

> 本文档遵循 `codex-workflow-constants.md`。
> 触发条件：用户提出调研/对比/选型/搜索类问题。
> 入口：从 `CODEX.md` 路由到此。

---

## 核心原则

- 先证据后结论
- 至少两类独立来源交叉验证
- 结论必须绑定当前约束条件
- 输出可执行建议，不只给概念

---

## 调研分层

### 快速

- 适合边界清晰、低风险问题
- 执行 Phase 0-3

### 标准

- 适合技术选型或即将进入实现
- 执行 Phase 0-4，并产出报告

### 深度

命中任意一条：

- 影响架构主路径
- 方案分歧大且无法收敛
- 需要多轮迭代与多文档深读

---

## 流程阶段

### Phase 0：问题定义

输出研究卡片：

- 研究问题
- 技术栈与硬约束
- 目标输出
- 截止时间

### Phase 1：快速检索

- 官方文档
- 社区实践
- 方案对比材料

输出：候选方案列表与待深挖问题。

### Phase 2：深度研读

Codex 对候选方案输出：

- 核心优势
- 已知限制
- 适用场景
- 不适用场景
- 与当前约束的匹配度

### Phase 3：结论收敛

- 形成对比矩阵
- 给推荐方案与排除理由
- 给风险与回退策略
- 需要时请 Claude Code 做交叉复核

### Phase 4：报告归档（可选）

写入 `docs/research/YYYY-MM-DD-[topic].md`：

- 背景与约束
- 方案对比
- 推荐结论
- 实施建议
- 风险与回退

---

## 输出格式

对用户输出必须包含：

- 推荐方案（1 个）
- 备选方案（至少 1 个）
- 不采用理由
- 证据来源
- 落地步骤（可执行）

---
