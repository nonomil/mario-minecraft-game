# BOSS 与门禁战学习融合实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在不破坏 BOSS 与门禁战动作节奏的前提下，把学习融合进战前准备、战中读懂提示与战后复盘奖励，让高光玩法既更有学习性也更像游戏。

**Architecture:** 方案分三层落地：战前微学习、战中轻语义提示、战后词汇复盘。第一阶段不做“高压战中频繁弹题”，而是优先复用现有挑战、村庄、Boss HUD、掉落与图鉴入口，先把最小可感知闭环做顺。

**Tech Stack:** HTML5、CSS3、JavaScript (ESM)、现有 `12-challenges.js` / `12-village-challenges.js`、现有 `15-entities-boss.js` / `15-entities-boss-dragon.js`、现有 UI/HUD 与掉落系统

## 1. 目标体验

第一阶段完成后，玩家应该能明显感受到：

1. 打 BOSS 前会做一个很短的“准备”，但不会觉得被上课。
2. BOSS 战中出现的词不是装饰，而是真的能帮助理解机制。
3. 打完 BOSS 后，系统会告诉玩家“这场学到了什么”。
4. 学习结果会影响奖励、图鉴或后续成长，而不是只变成抽象分数。

## 3. 风险与控制

### 3.1 风险：BOSS 战被学习弹窗打断

**控制方式：**
- 战中不做高频弹窗
- 显性微学习只放在战前
- 阶段提示优先 HUD / banner

### 3.2 风险：提示过多，Boss HUD 变吵

**控制方式：**
- 只保留高频核心词
- 一次只突出一个重点词
- 保持显示时长短

### 3.3 风险：奖励过强破坏 Boss 平衡

**控制方式：**
- 开局 Buff 只做轻增益
- 宝箱只做轻加权
- 不做强保底传奇收益

## 4. 完成标准

满足以下条件，可视为 BOSS 学习融合第一阶段基本完成：

1. 门前微学习可稳定触发。
2. 阶段目标词能帮助理解当前目标。
3. 技能预警词能帮助判断高频动作。
4. 战后词汇复盘能形成奖励承接。
5. 村庄能承担基础 BOSS 训练入口。
6. BOSS 战节奏没有被明显拖慢。

---

## 5. 建议执行策略

推荐顺序：

1. 先搭 BOSS 学习配置层
2. 再做门前微学习
3. 再做阶段目标词
4. 再做技能预警词
5. 再做战后复盘
6. 最后接村庄训练入口

这样做的原因：

- 先把配置统一，后续不容易返工
- 先完成最容易被玩家感知的门前收益
- 再逐步把学习渗透到战中和战后
