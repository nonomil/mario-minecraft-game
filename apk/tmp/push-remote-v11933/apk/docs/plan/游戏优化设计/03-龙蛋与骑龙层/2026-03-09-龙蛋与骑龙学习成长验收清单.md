## 1. 本清单目的

本清单用于验证：

> 龙系统中的学习性，是否已经从“战后附带奖励”升级为“可持续承接学习成果的长期成长层”。

重点不再只看“龙有没有新奖励”，而要看：

1. 学习结果是否真的能稳定喂给龙成长。
2. 龙成长是否对玩家可见、可理解、可期待。
3. 骑龙词汇暴露是否自然，不打断操作。
4. 龙训练场是否成为低压学习入口。
5. 当前已有召唤龙 / 骑龙主玩法是否保持兼容。

---

## 2. 验收范围

本轮只覆盖龙系统学习成长层：

- 龙蛋唤醒进度
- 龙成长值 / 技能解锁
- 骑龙 HUD 词汇暴露
- 龙训练场
- 龙图鉴 / 收藏承接
- 与普通流程 / BOSS 战后奖励的接口

本轮不作为实现目标的内容：

- 重做现有召唤龙 / 骑龙基础功能
- 重做龙战动作系统
- 完整龙形态、龙皮肤、全量特效扩展
- 骑龙高压战斗中的高频显性测验

## 4. 功能验收项

## 4.1 龙学习成长统一性

**目标：** 龙成长不再只靠单一来源，而能承接多个学习事件。

- [ ] 新词收集可驱动龙成长
- [ ] 普通挑战或村庄训练可驱动龙成长
- [ ] BOSS 战后复盘可驱动龙成长
- [ ] 龙训练完成可驱动龙成长
- [ ] 不同来源的成长结果口径一致

**通过标准：**

- 学习结果不再只流向分数，而是能稳定流向龙成长层

## 4.2 龙蛋唤醒进度可视化

**目标：** 玩家能感知“龙正在被养成”。

- [ ] 龙蛋存在清晰进度表达
- [ ] 阶段变化可见，例如发光、裂纹或状态变化
- [ ] 单次成长提示不会刷屏
- [ ] 玩家能理解本次成长来自哪类学习事件
- [ ] 首次阶段变化能在前期较早出现

**通过标准：**

- 玩家知道“这次学习让龙更接近下一阶段”

## 4.3 龙成长与解锁结果

**目标：** 龙成长结果不是抽象数值，而有实际承接。

- [ ] 至少一种成长值被稳定记录
- [ ] 至少一种解锁结果可见
- [ ] 解锁结果与学习主题存在合理对应
- [ ] 第一阶段解锁以增强 / 展示为主，不强锁核心主功能
- [ ] 旧存档或旧玩家路径兼容正常

**通过标准：**

- 玩家能说清“为什么龙成长了”“成长后有什么变化”

## 4.4 骑龙词汇暴露与动作绑定

**目标：** 骑龙时的词汇能帮助记忆动作，而不影响操作。

- [ ] 核心词汇与实际动作一一对应
- [ ] HUD 提示为短词或图标 + 短词
- [ ] 强提示不会频繁连刷
- [ ] 高压场景词汇暴露有降频或弱化
- [ ] 初次接触新词时更明显，熟词时更克制

**通过标准：**

- 玩家能通过操作自然记住词，而不是被强迫读词

## 4.5 龙训练场

**目标：** 龙训练场是低压练习区，不是长测验区。

- [ ] 存在清晰训练入口
- [ ] 训练内容围绕动作词 / 方向词 / 技能词
- [ ] 单次训练时长可控
- [ ] 可重复进入
- [ ] 训练结果能回流到龙成长或图鉴

**通过标准：**

- 玩家会把龙训练理解为“练技能 + 学词”的小游戏

## 4.6 龙图鉴 / 收藏承接

**目标：** 龙线学习成果能沉淀为长期收藏。

- [ ] 至少一类龙主题词进入图鉴或收藏
- [ ] 解锁条件清晰可理解
- [ ] 图鉴 / 收藏不是纯摆设，能体现成长进度
- [ ] 收藏节奏不过快，也不过慢

**通过标准：**

- 玩家能感知龙线学习成果被长期保存

## 4.7 低年级与通用负担控制

**目标：** 龙线学习成长增强趣味，不增加额外挫败。

- [ ] 前期第一次成长反馈出现较早
- [ ] 骑龙 HUD 不会因词汇暴露显得过吵
- [ ] 龙训练单次时长建议控制在 `45 ~ 75s`
- [ ] 第一阶段核心词汇量控制在 `5 ~ 6`
- [ ] 单次强提示建议约 `1.0s`

**通过标准：**

- 小学低年级玩家也能感知成长乐趣，而不会被额外说明压垮

## 5. 回归兼容项

以下内容不属于本轮新增目标，但必须确认不被破坏：

- [ ] 现有召唤龙流程正常
- [ ] 现有骑龙控制正常
- [ ] 龙相关 BOSS / 战斗流程正常
- [ ] 普通流程学习基础层正常
- [ ] BOSS 战后复盘与奖励层正常
- [ ] HUD 基础信息没有被龙线提示覆盖

---

## 6. 推荐验收流程

建议按下面顺序验：

1. 先验普通流程学习事件是否能推动龙进度
2. 再验龙蛋阶段反馈是否清晰
3. 再验一次骑龙词汇暴露与操作绑定
4. 再验龙训练场
5. 最后验图鉴 / 收藏与跨系统承接

建议覆盖的最小场景：

- 场景 A：收集若干词汇，检查龙进度是否变化
- 场景 B：完成一次村庄训练，检查龙成长反馈
- 场景 C：骑龙进行基础动作，检查 HUD 词汇暴露
- 场景 D：完成一次龙训练，检查成长与收藏回流
- 场景 E：完成一次 BOSS 战后复盘，检查龙成长接口

## 8. 通过门槛

- [ ] 至少三类学习事件可驱动龙成长
- [ ] 龙蛋进度具备阶段可视化反馈
- [ ] 至少一种龙成长解锁结果可见
- [ ] 骑龙词汇暴露与动作绑定成立
- [ ] 龙训练场可在低压下重复使用
- [ ] 龙图鉴 / 收藏至少有一类承接打通
- [ ] 已存在召唤龙 / 骑龙主流程兼容正常
