# 游戏优化实施计划 - 版本路线图

- **当前版本**: v1.3.2
- **已实现**: 基础游戏循环、6种盔甲、双傀儡召唤、9种敌人、8种生物群系、宝箱掉落系统、食物回血系统、背包点击交互（食物回血+盔甲装备）、召唤系统重构（背包点击召唤）

## 阶段划分

### 第一阶段：背包与物品系统 (v1.3.x)
优先级最高。完善物品使用机制，让已有的掉落物品真正可用。
- 风险低，改动范围小
- 每个版本独立可测试

### 第二阶段：BOSS战系统 (v1.4.x)
核心玩法扩展。先搭框架，再逐个添加BOSS。
- v1.4.0 是关键版本，需要搭建BOSS战的完整框架
- 后续BOSS复用框架，开发速度快

### 第三阶段：环境系统升级 (v1.5.x)
丰富游戏世界。改进现有环境，新增末地。
- 每个环境独立，互不影响
- 末地为可选内容

## 每个版本的工作流程

```
1. 阅读对应版本的计划文档
2. 按文档中的修改清单逐项实施
3. 本地浏览器测试 (Game.html)
4. 更新 package.json 版本号
5. 更新 CHANGELOG.md
6. git commit + git push
7. 等待 CI 构建 APK
8. 真机测试验证
```
