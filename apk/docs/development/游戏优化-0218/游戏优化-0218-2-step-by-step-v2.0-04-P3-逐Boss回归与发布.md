# v2.0-P3：逐BOSS回归与发布清单

## 0. 目标

确保 P0~P2 改动后，四个BOSS和群系系统稳定可发布。

## 1. Step 1：逐BOSS回归矩阵

覆盖对象：
1. Wither（2000分）  
2. Ghast（4000分）  
3. Blaze（6000分）  
4. Wither Skeleton（8000分）

每个BOSS检查项：
1. 触发：按分数正确进入战斗。  
2. 可见：本体、弹幕、受击特效都可见。  
3. 可打：近战/远程至少一条稳定击杀路径。  
4. 边界：玩家不越墙，相机锁可恢复。  
5. 结算：胜利奖励与退场正常。

## 2. Step 2：群系回归矩阵

检查项：
1. 森林 -> 樱花 -> 雪地 -> 沙漠 -> 山地 -> 海洋 -> 火山/地狱 链路连续。  
2. `minStay` 生效，无异常卡住。  
3. `stepScore` 调整后，切换频率符合预期。  
4. 高温环境伤害节奏不异常（不突发、不卡死）。

## 3. Step 3：异常路径回归

检查项：
1. BOSS战中暂停/恢复。  
2. BOSS战中死亡/复活。  
3. 低箭矢开战。  
4. 长时间游玩（>=10分钟）后的相机与渲染稳定性。

## 4. Step 4：发布记录模板

发布记录至少包含：
1. 变更文件列表  
2. P0/P1/P2 验收截图或日志  
3. 已知遗留问题  
4. 回滚策略（按模块回滚）

## 5. 通过标准（P3 DoD）

1. 四个BOSS全部通过“可见+可打+可结算”。  
2. 群系节奏通过“不短不长”主观验收。  
3. 无新增致命Bug（卡死、黑屏、不可恢复锁相机）。
