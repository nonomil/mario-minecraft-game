# 版本修订记录（Progress）

> 目的：记录每个版本的**变更摘要**与**构建信息**，并与 APK 版本号保持一致。

## 版本号命名规范
采用语义化版本：`MAJOR.MINOR.PATCH`
- **MAJOR**：重大版本（玩法/系统级改造、架构变更、资源大规模调整）
- **MINOR**：功能新增与体验提升（新敌人/新机制/新模式等）
- **PATCH**：修复与小优化（bug 修复、轻微 UI 调整、参数微调）

> 对应关系：
> - **Android**：`versionName` = 语义版本；`versionCode` 递增（每次发布 +1）
> - **本记录**：每次发布必须新增一条版本记录，保持与 `versionName` 一致

## 模板（复制使用）
```
## vX.Y.Z  （发布日期：YYYY-MM-DD）
- 类型：MAJOR / MINOR / PATCH
- APK 版本：versionName = X.Y.Z，versionCode = N
- 构建方式：本地 / CI（GitHub Actions）
- 主要变更：
  - ...
  - ...
- 兼容性说明：
  - 需要卸载重装？否（同签名 + versionCode 递增）
- 备注：
  - ...
```
