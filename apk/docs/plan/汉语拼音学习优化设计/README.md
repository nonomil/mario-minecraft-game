## 1. 这个目录现在怎么用

这个目录已经不是单纯的脑暴区，而是：

- 有方向
- 有架构
- 有数值
- 有第一阶段开工入口

但它**不适合整包并行执行**。

当前更合理的用法是：

1. 先看执行导图，判断现在做哪一层
2. 只启动幼儿园模式第一阶段最小闭环
3. 其他专题保留为后续补强文档

## 3. 当前执行结论

一句话结论：

> 这条线值得做，但当前只建议执行“第一阶段最小闭环”，不要把桥接、语音识别、家长面板一起开。

推荐现在就做的范围：

- `K-M1` 听音认物
- `K-M2` 村庄拼音小游戏
- `K-M3` 贴纸册 / 小龙孵化反馈

推荐暂缓的范围：

- 语音识别主玩法
- 家长控制面板完整产品化
- 双语桥接模式正式落地
- 大规模内容铺量

## 6. 推荐实现顺序

建议按下面顺序推进：

1. 模式配置骨架
2. 听音认物闭环
3. 村庄拼音小游戏
4. 贴纸 / 小龙反馈
5. 内容量级与词库结构补齐
6. 儿童测试与家长入口规划
7. 双语桥接模式

换句话说：

- 先验证“低龄玩法成立”
- 再补“内容和产品层”
- 最后做“桥接和扩展层”

## 8. 与主项目的关系

这个目录里的方案不主张：

- 做两套完全分裂的游戏代码
- 让幼儿园模式直接复用小学英语主交互

更推荐：

- 共享底层学习事件
- 共享配置驱动框架
- 分离内容层与交互层

这意味着它更像：

> 在当前项目里新增一个低龄学习层，而不是另起一个新项目。
