﻿# 普通流程学习融合实施计划（基础层，2026-03-09 校准版）

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在不改动 BOSS / 门禁战 / 龙蛋 / 骑龙专用学习系统的前提下，把普通游戏流程中的学习事件统一成“收集、理解、训练、复盘、奖励、收藏”闭环，并优先服务小学低年级。

**Architecture:** 方案以“统一普通学习事件模型 + 复用现有挑战/村庄/收词/UI/宝箱/图鉴入口”为核心。第一阶段重点不是增加更多弹窗题，而是把普通流程里的学习事件重新组织成低压显性学习、中压轻量学习和低负担成长承接。

**Tech Stack:** HTML5、CSS3、JavaScript (ESM)、现有挑战系统、现有村庄挑战系统、现有 UI/HUD、现有掉落与宝箱逻辑

---

## 0. 计划前提

### 已确认的产品结论

1. 本计划只负责普通流程学习融合基础层。
2. 主目标用户仍为小学低年级（1–3 年级）。
3. 普通流程优化不应再默认等于“增加更多弹窗题”。
4. BOSS / 门禁战学习融合与龙蛋 / 骑龙学习成长，由独立专题文档承接。

### 本计划包含

- 普通学习事件统一抽象
- 收词后的语义强化
- 普通流程即时反馈
- 宝箱 / 收藏 / 图鉴对学习结果的承接

### 第一阶段推荐默认值

实现时如无更强证据，默认采用以下建议值：

- `challengeFrequency = 0.20`
- `wordCardDuration = 1100ms`
- 普通挑战 `timeLimit = 18s`
- 普通挑战答错惩罚 `-4 分`
- `Word Gate spawnChance = 0.12`
- `Word Gate minScore = 240`
- 复活学习 `4 题 / 35s / 3 题正确`
- 村庄训练 `6 题 / 60~90s`

## 1. 目标体验

普通流程第一阶段完成后，玩家应有以下感受：

1. 学习不只发生在答题弹窗里，而是在收集、探索、训练、复盘中持续发生。
2. 学会词汇后，不仅得到分数，还会推进宝箱、收藏、图鉴或后续收益。
3. 村庄不只是挑战点，而是低压训练和复盘中心。
4. 小学低年级玩家在前 5~10 分钟更容易进入状态，不会被频繁弹窗或惩罚压退。

## 2. 推荐实现顺序

### Task 1：普通学习事件统一抽象层

**Files:**
- Modify: `src/modules/12-challenges.js`
- Modify: `src/modules/12-village-challenges.js`
- Modify: `src/modules/09-vocab.js`
- Modify: `src/modules/01-config.js`

**目标：**
把普通流程中的学习行为统一抽象为“普通学习事件”，避免后续奖励和收藏只能围绕答题结算挂接。

**Step 1: 定义普通学习事件分类**

第一版至少覆盖：
- 收到新词
- 完成普通挑战
- 通过 Word Gate
- 完成村庄训练
- 完成一次轻复盘

**Step 2: 统一事件结果对象**

结果对象至少包含：
- `eventType`
- `source`
- `wordKey` 或 `themeKey`
- `successLevel`
- `rewardEligible`
- `uiHintType`

**Step 3: 为后续模块提供统一读入口**

让奖励、宝箱、图鉴、收藏、村庄训练都能消费同一套学习事件。

**Step 4: 最小验证**

验证：
- 新旧普通挑战仍可正常运行
- 收词与村庄挑战可接入统一事件结构
- `challengeFrequency` 下调后，普通流程不会高频打断

### Task 2：收词与场景语义强化

**Files:**
- Modify: `src/modules/12-challenges.js`
- Modify: `src/modules/09-vocab.js`
- Modify: `src/modules/10-ui.js`
- Modify: `Game.html`

**目标：**
把“收词 = 播音 + 词卡”的旧逻辑，升级为“收词 = 暴露 + 语义强化 + 后续承接”的入口。

**Step 1: 为收词增加主题 / 场景标签**

让收词时能够知道：
- 当前群系
- 当前主题
- 当前可能关联的敌人 / 材料 / 任务

**Step 2: 加入轻量场景提示**

提示应短，不做额外大弹窗。  
优先采用：
- HUD 短提示
- 词卡上的一条上下文信息
- 收藏 / 图鉴进度提示

**Step 3: 控制信息量**

要求：
- 单次提示短
- 不连续堆叠多层提示
- 不影响跑图节奏
- 语义提示默认长度控制在 `8 ~ 18` 个汉字或同等短英文量级

**Step 4: 最小验证**

验证：
- 收词后仍流畅
- 玩家能明显感知词与场景的联系

### Task 3：普通流程即时反馈层

**Files:**
- Modify: `src/modules/13-game-loop.js`
- Modify: `src/modules/10-ui.js`
- Modify: `src/modules/01-config.js`

**目标：**
把普通学习事件转化成“可见但不打断”的即时反馈。

**Step 1: 定义普通流程反馈类型**

第一版建议包括：
- 小幅资源奖励
- 临时探索收益
- 收藏 / 图鉴提示
- 训练进度提示

**Step 2: 区分高频与低频反馈**

要求：
- 高频事件只给轻提示
- 低频或关键达成事件才给更明显反馈

**Step 3: UI 反馈最小化**

反馈应做到：
- 可见
- 不遮挡
- 儿童可理解

**Step 4: 最小验证**

验证：
- 普通流程里学习反馈可见
- 不会因为反馈太多而打断探索
- 强提示之间默认保留约 `6s` 以上间隔

**Files:**

把村庄升级为普通流程中的低压学习中枢。

**Step 1: 重新定义村庄的普通流程角色**

第一阶段至少承接：
- 主题词训练

- 段后轻复盘

**Step 2: 控制训练长度**

村庄训练不宜太长，重点是：
- 低压
- 可恢复

- 题量 `6 题`
- 单次训练时长 `60 ~ 90s`

复盘可先包含：
- 本段接触词汇

**Step 4: 最小验证**

验证：
- 村庄不再只是一次性挑战点
- 玩家能把村庄理解为“整理和准备”的地方
- 单次村庄训练不会超过约 `90s`

控制普通流程中的阅读负担、提示长度和弹窗频率，让低年级玩家更容易进入状态。

**Step 1: 调整频率而非只调分数**

重点控制：
- 弹窗频率
- 单次文本长度
- 高压时段提示数量

第一阶段默认目标：
- 普通挑战频率从 `0.30` 调整到 `0.20`
- 普通挑战限时从 `15s` 调整到 `18s`
- 单词卡停留从 `900ms` 调整到 `1100ms`
- 普通挑战答错惩罚从 `-8` 调整到 `-4`

**Step 2: 文案简化**

优先使用：
- 短句
- 高频词
- 图标 + 文本组合

**Step 3: 保留挑战性**

要求：
- 不把普通流程做成完全无判断
- 只是让输入压力更小、恢复更快

**Step 4: 最小验证**

验证：
- 前 5~10 分钟更顺
- 仍然保留基本参与感与成长感

## 3. 风险与控制

### 3.1 风险：提示过多，普通流程变吵

**控制方式：**
- 高频事件只给轻提示
- 关键达成事件才给强提示
- 限制同屏同时出现的信息量

### 3.2 风险：普通流程学习仍然过度依赖弹窗

**控制方式：**
- 明确区分显性学习与隐性学习
- 优先做收词语义强化、收藏承接、村庄复盘
- 不把所有学习设计都做成题目

### 3.3 风险：普通流程基础层与 BOSS / 龙专题混淆

**控制方式：**
- 本计划只覆盖普通流程
- BOSS / 龙相关需求全部转交 2026-03-09 专题文档

## 4. 完成标准

满足以下条件，可视为普通流程学习融合基础层基本完成：

1. 普通学习事件不再只等于答题事件。
2. 收词能承接语义强化，而不只是短暂词卡。
3. 宝箱 / 收藏 / 图鉴能承接普通学习结果。
4. 村庄能承担训练与轻复盘角色。
5. 小学低年级前期阅读负担和弹窗压力明显下降。
6. 不破坏现有 BOSS / 门禁战 / 龙蛋 / 骑龙玩法。

---

## 5. 建议执行策略

推荐顺序：

1. 先完成普通学习事件统一抽象
2. 再完成收词与场景语义强化
3. 再做普通流程即时反馈
4. 再做宝箱 / 收藏 / 图鉴绑定
5. 再做村庄训练与轻复盘
6. 最后做低年级负担控制与回归

这样做的原因：

- 先统一入口，后续不容易返工
- 先提升“自然发生的学习”
- 再补奖励承接与长期积累
- 最后再依据数值建议表做频率和压力微调
