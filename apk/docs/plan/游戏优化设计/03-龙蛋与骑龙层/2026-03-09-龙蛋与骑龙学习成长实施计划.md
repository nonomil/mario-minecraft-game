# 龙蛋与骑龙学习成长实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在不破坏当前已存在的龙蛋、召唤龙、骑龙高光玩法前提下，把学习成果稳定接入龙系统，形成“学习 → 龙成长 → 新能力 / 新展示 → 继续学习”的长期闭环。

**Architecture:** 第一阶段采用“叠加式成长层”而不是“硬门槛重构”。保留当前主项目已有的召唤与骑龙核心体验，把学习性优先接入龙蛋唤醒进度、龙成长条、骑龙词汇暴露、龙训练场与龙图鉴承接。

**Tech Stack:** HTML5、CSS3、JavaScript (ESM)、现有 `13-game-loop-dragon-controls.js`、现有 `15-entities-boss-dragon.js`、现有 `12-village-challenges.js`、现有 UI/HUD、掉落、图鉴与挑战系统

---

## 0. 计划前提

### 已确认的产品结论

1. 当前项目已经存在龙蛋、召唤龙、骑龙相关能力，第一阶段不建议为了学习系统而回退已有功能。
2. 龙系统学习成长层的职责是承接长期成长，不替代普通流程基础层与 BOSS 高光层。
3. 第一阶段重点是“让学习成果喂养龙成长”，而不是“再造一套独立考试系统”。
4. 骑龙中的学习设计应以轻量暴露、动作绑定、持续陪伴为主，不以高频弹窗测验为主。

### 本计划包含

- 龙学习成长统一配置与状态层
- 龙蛋唤醒进度与学习事件接入
- 龙经验 / 亲密度 / 技能解锁映射
- 骑龙 HUD 与控制词汇暴露
- 村庄中的龙训练场
- 龙图鉴 / 收藏 / BOSS 奖励承接

### 本计划不包含

- 重做现有骑龙主操作
- 重做现有龙战与 BOSS 数值平衡
- 一次性做完整龙形态、龙皮肤、全量特效
- 在骑龙高压战斗中增加高频显性测验

## 1. 目标体验

第一阶段完成后，玩家应该明显感受到：

1. 学习行为不再只是拿分，而会“喂给龙”形成成长反馈。
2. 龙蛋、龙状态、龙技能与学习进度之间有清晰因果。
3. 骑龙时看到的词汇是真正和动作、方向、技能强绑定的。
4. 龙系统成为长期留存目标，而不是一次性奖励展示。

### Task 3：龙经验 / 亲密度 / 解锁映射

**Files:**
- Modify: `src/modules/15-entities-boss-dragon.js`
- Modify: `src/modules/13-game-loop-dragon-controls.js`

**目标：**
把学习成果进一步沉淀为龙成长层的长期积累，而不是只停留在龙蛋唤醒阶段。

**Step 1: 明确三类成长值**

第一版建议：

**Step 2: 定义第一批解锁结果**

第一阶段建议优先解锁：
- 新 HUD 词汇显示
- 新训练主题
- 龙特效轻变化
- 轻量技能强化或冷却优化

**Step 3: 避免直接锁死现有功能**

要求：
- 不因学习不足而关闭已有召唤 / 骑龙
- 学习成长主要带来“增强与扩展”
- 把强门槛推迟到后续独立版本讨论

**Step 4: 最小验证**

验证：
- 成长值增长节奏稳定
- 解锁点可感知
- 不影响当前龙系统核心可玩性

## 3. 风险与控制

### 3.1 风险：为了学习成长反向破坏现有龙玩法

**控制方式：**
- 第一阶段不关掉已存在的核心能力
- 学习层只做叠加增强
- 旧存档优先兼容

### 3.2 风险：骑龙 HUD 词汇太多，影响操作

**控制方式：**
- 只保留高频核心词
- 强提示只给新词
- 高压时段自动降频

### 3.3 风险：龙成长节奏太慢，玩家感知不到

**控制方式：**
- 前两个阶段阈值拉近
- 先让玩家更快看到第一次变化
- 奖励以可视化反馈优先

### 3.4 风险：龙系统过早做成大而重的养成系统

**控制方式：**
- 第一阶段只做轻量成长层
- 不一次性铺满全部外观 / 技能 / 图鉴
- 先确认长期驱动有效，再扩展深度

---

## 4. 完成标准

满足以下条件，可视为龙系统学习成长第一阶段基本完成：

1. 普通学习事件能稳定驱动龙蛋或龙成长进度。
2. 学习成果能转换为至少一种龙成长结果。
3. 骑龙 HUD 中的词汇暴露能和动作形成自然绑定。
4. 村庄存在低压龙训练入口。
5. 龙图鉴或收藏系统能承接一部分学习结果。
6. 当前已存在的召唤龙 / 骑龙主流程未被明显破坏。

---

## 5. 建议执行策略

推荐顺序：

1. 先搭龙学习配置与状态层
2. 再接龙蛋唤醒进度
3. 再做成长值与轻解锁
4. 再做骑龙词汇暴露
5. 再接龙训练场
6. 最后接龙图鉴与跨系统奖励

这样做的原因：

- 先统一数据结构，后续不容易返工
- 先让玩家看到“学习会养龙”，动机建立最快
- 再逐步把龙成长扩到 HUD、训练与收藏
