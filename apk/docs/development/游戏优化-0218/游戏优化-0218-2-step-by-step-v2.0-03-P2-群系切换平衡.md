# v2.0-P2：群系切换平衡（不短不长）

## 0. 目标

让群系切换节奏更平衡，重点解决火山/地狱停留偏短。

## 1. Step 1：参数调优（第一轮）

涉及文件：
- `config/biomes.json`

建议首轮参数：
1. `switch.stepScore`：`200 -> 300`  
2. `switch.minStay.volcano`：`{score:200,timeSec:40} -> {score:320,timeSec:70}`  
3. `switch.minStay.nether`：`{score:250,timeSec:45} -> {score:360,timeSec:75}`  
4. 其他群系先保持，减少扰动。

## 2. Step 2：解锁分数间距拉开（第二轮）

涉及文件：
- `config/biomes.json`

建议：
1. 低阶群系维持较密（保持新手反馈）。  
2. 中后期群系增大间距（减少高压群系过早、过频轮换）。  
3. 火山与地狱解锁点适度后移（根据测试数据再定）。

## 3. Step 3：设置项下限保护

涉及文件：
- `Game.html`
- `src/modules/16-events.js`
- `src/modules/09-vocab.js`

操作：
1. 将 `opt-biome-step` 的 `min` 从 `50` 提升到 `150` 或 `200`。  
2. `normalizeSettings()` 同步提高下限，避免历史配置穿透。  
3. 保存设置后即时生效，并更新UI反馈。

## 4. Step 4：观测指标

采集项（手工即可）：
1. 火山单次停留时长（秒）  
2. 火山单次停留增分（分）  
3. 10分钟内群系切换次数  
4. 玩家主观评分（太快/合适/太慢）

## 5. 验收标准（P2）

1. 火山/地狱明显不再“刚进入就切走”。  
2. 全局切换仍有节奏，不出现“长时间只在一个群系”。  
3. 设置项无法调到导致“闪切”的危险区间。
