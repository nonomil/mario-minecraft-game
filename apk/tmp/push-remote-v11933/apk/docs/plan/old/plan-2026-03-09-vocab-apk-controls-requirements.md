# 需求文档：APK词汇库切换修复 + 移动端控制优化

**创建时间**: 2026-03-09
**状态**: 需求讨论中

## 需求理解

### 问题1：APK版本词汇库切换不生效
- **现象**：网页版（Game.html）可以切换词汇库并生效，但编译的APK版本切换词汇库没有实际生效
- **影响范围**：Android APK版本
- **可能原因**：
  - LocalStorage在Capacitor环境下的行为差异
  - 配置同步问题
  - 词汇库加载逻辑在APK环境下的问题

### 问题2：移动端控制按键显示逻辑优化
- **期望行为**：
  - 地面状态：只显示左右两个按键（靠近地面位置）
  - 骑龙状态：在左右按键上方显示上下两个按键
  - 下龙后：上下按键消失
- **交互逻辑**：
  - 跳上龙 = 骑龙
  - 在龙上再跳一下 = 下龙

## 歧义点与待确认问题

### 问题1相关
1. **词汇库切换入口**：APK版本中，用户在哪里切换词汇库？是否有UI界面？
2. **切换后的表现**：切换后UI显示的词汇库名称是否改变？还是UI改变了但实际游戏中的单词没变？
3. **测试方式**：如何验证词汇库是否真正生效？（比如不同词汇库的单词内容不同）
4. **存储位置**：网页版使用LocalStorage，APK版本是否也应该用LocalStorage？还是需要用Capacitor的Preferences API？

### 问题2相关
1. **"靠近地面位置"**：具体指屏幕的什么位置？底部多高的区域？
2. **按键尺寸**：上下按键的尺寸是否与左右按键相同？
3. **按键间距**：上下按键与左右按键之间的垂直间距是多少？
4. **过渡动画**：上下按键出现/消失是否需要动画效果？还是直接显示/隐藏？
5. **其他状态**：玩家在其他坐骑上（如果有）是否也需要类似的控制逻辑？
