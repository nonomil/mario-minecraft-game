# BOSS战优化 Step-by-Step 计划

## 目标
- 提升可学习性与战斗节奏
- 明确阶段化与反馈
- 增加风险/收益决策

## 验收标准（通用）
- 进入战斗后 3~5 秒内能看到明确“提示/节奏”
- 任何高伤害招式都有可识别预警
- 有至少 1 个明确的“惩罚窗口”（玩家可反击）
- 调试入口可稳定复现

## BOSS：Evoker

### 设计要点
- 召唤小怪 + 护盾时机
- 决策分支：清怪 vs 打 Boss

### Step-by-Step
1. 现状梳理
   - 召唤逻辑与频率
2. 护盾机制
   - 护盾期间召唤小怪
   - 护盾破裂后短输出窗口
3. 召唤节奏
   - 每波 2~4 只
   - 波次间隔可控
4. 调试接口
   - 触发召唤
   - 护盾开关
5. 测试
   - 清怪/打 Boss 选择有意义
   - 护盾窗口可控

### 风险点
- 小怪数量过多导致资源浪费
