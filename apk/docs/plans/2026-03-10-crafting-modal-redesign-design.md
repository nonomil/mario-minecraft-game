# 合成模式菜单重构设计（方案 C）

**日期**: 2026-03-10  
**目标**: 解决“合成选择后卡住”的交互问题，并将合成菜单升级为混合模式（配方一键填充 + 材料多选）。

## 现状问题与根因
- 现象: 关闭合成面板后仍阻挡点击，表现为“卡住”。
- 根因: CSS 只对 `#inventory-modal` 定义了显示/隐藏，`#crafting-modal` 仅有 `class="inventory-modal"` 却未继承规则，导致 `display: block` 与 `pointer-events: auto` 始终生效。

## 设计原则
- 维持像素风 UI 视觉一致性，避免引入与当前风格冲突的组件。
- 支持“新手一键合成”与“熟练手动调整”双路径。
- 反馈清晰：配方匹配、缺少材料、多选材料即时提示。

## 信息架构（方案 C）
- 配方区: 可点击配方卡，一键填充材料。
- 已选材料栏: 芯片形式展示材料与数量，支持减量。
- 材料网格: 每个材料卡带 `+ / −`，支持多选数量。
- 产出与状态区: 显示目标配方、缺少/多选提示，合成按钮仅在匹配时可用。
- 操作按钮: `清空选择`、`自动填充`、`开始合成`。

## 交互细节
- 点击配方卡: 自动填充材料，并高亮配方。
- 点击材料卡: 等价于 `+1`；`+ / −` 细粒度调节。
- 已选材料芯片: 点击 `−` 直接减量。
- 状态提示: 
  - 完全匹配时提示“已匹配配方，可合成”
  - 未匹配时显示“缺少/多选材料”

## 非目标
- 不新增配方或材料类型。
- 不变更合成逻辑与物品掉落规则。

## 受影响文件
- `src/styles/20-touch-controls.css`
- `Game.html`
- `src/modules/01-config.js`
- `src/modules/16-events.js`
- `src/modules/13-game-loop.js`

## 验收标准
- 关闭合成面板后不再阻挡点击（无“卡住”）。
- 配方一键填充可用，材料 `+ / −` 可用。
- 匹配/缺少/多选提示明确，合成按钮状态正确。
