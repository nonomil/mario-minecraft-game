# 开局弹窗与档案选择流程设计

日期：2026-02-23
范围：仅 `apk/` 目录，首页弹窗与账号/档案选择。

## 目标
- 开局说明弹窗只保留两个按钮：左“跳过”、右“选择档案”。
- 档案选择页底部新增“点击确定”按钮。
- 解决“请先选择档案”按钮无效的问题，明确按钮跳转与校验逻辑。

## 需求摘要
- 页面 1（开局说明弹窗 `#screen-overlay`）：
  - 左按钮：跳过（直接进入游戏）。
  - 右按钮：选择档案（关闭 overlay，打开登录/档案选择）。
  - 隐藏/移除排行榜与复活等按钮。
- 页面 2（账号/档案选择 `#login-screen`）：
  - 底部“点击确定”：若已选账号或已输入用户名 -> 进入游戏并关闭登录页。
  - 若未选择/输入：提示并留在本页。

## 交互流程
1. 进入游戏 -> 显示 `#screen-overlay`。
2. 点“跳过” -> 关闭 overlay -> 直接开始游戏。
3. 点“选择档案” -> 关闭 overlay -> 打开 `#login-screen`。
4. 在 `#login-screen` 选择账号或输入用户名 -> 点“点击确定” -> 进入游戏。
5. 若未选择/输入 -> “点击确定”提示校验错误。

## 结构改动
- `apk/Game.html`：
  - `#screen-overlay` 的按钮区改为双按钮布局（跳过/选择档案）。
  - `#login-screen` 底部新增“点击确定”按钮。

## 样式
- 双按钮左右对齐、等宽；右侧主按钮强调色。
- “点击确定”置底居中；禁用态低饱和灰色。

## 风险与约束
- 不影响既有 overlay 逻辑（暂停/复活等其他场景）。
- 登录页无选择时需提示但不阻塞输入。

## 测试要点
- 开局说明弹窗显示双按钮。
- “跳过”可直接进入游戏。
- “选择档案”可进入登录页。
- 登录页“点击确定”在未选择/输入时提示。
- 登录页选择/输入后能进入游戏。
