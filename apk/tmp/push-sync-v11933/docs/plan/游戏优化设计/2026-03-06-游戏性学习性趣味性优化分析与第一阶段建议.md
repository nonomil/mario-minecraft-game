## 1. 本文档目的

这份文档重新定义“普通流程里的学习优化”，不再把它理解为“再增加更多挑战题或更多弹窗题”，而是回答下面 4 个问题：

1. 当前项目在 **普通游戏流程** 中，学习应该如何更自然地发生。
2. 哪些学习内容适合做成显性练习，哪些应该做成隐性学习。
3. 第一阶段普通流程学习优化到底先抓什么。
4. 普通流程学习与 BOSS / 门禁战 / 龙蛋 / 骑龙学习设计如何分层。

这里的“普通流程”明确指：

- 跑图
- 收词
- 普通学习挑战
- Word Gate
- 复活学习
- 村庄
- 宝箱与掉落
- 图鉴 / 收藏 / 复盘

不包含：

- BOSS 战内专用学习设计
- 门禁战前微学习专用设计
- 龙蛋与骑龙长期学习成长

## 2. 校准后的核心判断

结合 2026-03-09 的外部案例调研，普通流程学习优化需要从“题目驱动”升级为“场景驱动 + 奖励驱动 + 成长驱动”。

### 2.1 之前判断里仍然成立的部分

以下判断继续保留：

1. 当前项目不缺学习内容，缺的是更顺的学习闭环。
2. 第一阶段依然适合先服务小学低年级。
3. 奖励闭环依然是普通流程里的第一优先。
4. 降低挫败依然非常重要。

### 2.2 需要修正的部分

旧思路里最大的问题不是“方向错了”，而是过于默认：

- 学习提升 = 更多挑战题
- 学习强化 = 更多词卡提示
- 学习奖励 = 主要来自答题结算

调研后的修正是：

- 学习提升不应只依赖弹窗题
- 学习强化应更多依赖场景语义、敌人语义、道具语义、任务语义
- 学习奖励来源应覆盖收集、互动、训练、复盘，而不是只覆盖答题

### 2.3 新的总原则

普通流程学习融合应遵循这 4 条原则：

1. **低压区适合显性学习**
   - 例如村庄、图鉴、复盘、训练场
2. **中压区适合轻量学习**
   - 例如跑图收集、轻交互判断、门前提示
3. **高压区不在本计划内展开**
   - BOSS / 骑龙另走独立专题文档
4. **学习要服务继续游玩**
   - 学会之后应带来理解优势、收集优势、奖励优势或成长优势

## 3. 普通流程学习的新分层模型

普通流程学习不再只按“题型”划分，而按**玩家压力等级**划分。

## 3.1 低压层：显性学习主场

适用场景：

- 村庄
- 图鉴 / 收藏
- 战后复盘
- 训练节点

适合做的事：

- 完整词汇练习
- 词义匹配
- 技能词预习
- 主题词复盘
- 图鉴解锁说明

设计原则：

- 可停下来学
- 可以稍长一点
- 更适合承接“理解”和“巩固”

## 3.2 中压层：轻量学习主场

适用场景：

- 普通跑图
- 收词
- 宝箱前后
- 普通怪与环境互动
- Word Gate
- 轻度精英前置提示

适合做的事：

- 词汇短暴露
- 一步判断
- 轻量选择
- 敌人 / 道具 / 环境语义理解
- 收词后的上下文强化

设计原则：

- 不要频繁暂停
- 单次信息量小
- 让玩家“边玩边懂”

## 3.3 高压边界：本计划不展开

适用场景：

- BOSS 战
- 门禁战核心战斗段
- 骑龙高压操作段

这部分不在本文件里展开，只在边界上明确：

- 不应简单复用普通流程弹题逻辑
- 应使用后续独立专题文档处理

## 4. 普通流程里真正该补的学习能力

## 4.1 收集不是“碰到就结束”，而应成为上下文强化入口

旧逻辑里，收词更偏向：

- 收到单词
- 播放发音
- 闪一下词卡

新逻辑里，收词应进一步承担：

- 与当前群系、敌人、材料、任务建立关联
- 给出短语义提示，而不是只显示单词本身
- 作为后续宝箱、图鉴、主题收藏的来源

例子：

- 在海洋群系捡到 `bubble`，词卡后接一条简短环境提示
- 在矿洞捡到 `torch`，提示其与黑暗环境的作用关联

## 4.2 场景记忆强化应改写为“语义理解强化”

旧文档里的“场景记忆强化”方向保留，但定义要改：

它不再只是：

- 多显示图片
- 多弹提示

而应升级为：

- 敌人词和敌人行为绑定
- 材料词和功能绑定
- 群系词和环境机制绑定
- 任务词和目标绑定

也就是：

> 让玩家不是“看见这个词”，而是“知道这个词在当前世界里是什么意思、有什么用”。

## 4.3 村庄应从“挑战点”升级为“低压学习中枢”

村庄是当前项目中最适合承接普通流程学习升级的节点。

村庄后续不应只提供：

- 一次挑战
- 一次奖励

而应逐步承担：

- 主题词训练
- 门前预习
- 战后复盘
- 图鉴查看
- 收藏兑换
- 龙与 BOSS 相关后续训练的入口

## 4.4 宝箱与收藏不只是奖励系统，也是学习承接系统

普通流程里最值得加强的一点是：

- 让学习结果更多流向“收集、收藏、解锁”

适合的方向：

- 某主题词收集达到阈值 → 解锁主题宝箱池
- 某类词理解度达标 → 解锁图鉴页
- 某次训练表现好 → 提高本轮探索收益

## 4.5 复盘应成为普通流程的固定后处理节点

普通流程里不一定每次都要做完整测验，但应该有更多“轻复盘”。

适合做成复盘的时机：

- 一段探索结束后
- 村庄训练结束后
- 一局失败或结算后

复盘内容适合包括：

- 本段接触词汇
- 当前主题词
- 哪些词已进入收藏 / 图鉴
- 下一步推荐学习方向

## 5. 第一阶段优先级（普通流程）

校准后，普通流程第一阶段建议按下面顺序推进。

### 5.1 第一优先：普通流程奖励闭环

重点不是“答题奖励”，而是“普通学习事件奖励”。

包括：

- 收词奖励
- 轻交互奖励
- 村庄训练奖励
- 复盘达成奖励

### 5.2 第二优先：普通流程语义强化

重点是：

- 敌人 / 道具 / 群系 / 任务的语义理解

而不是：

- 单独再加更多无上下文题目

### 5.3 第三优先：低压复盘与收藏承接

重点是：

- 村庄
- 图鉴
- 收藏册
- 战后 / 段后复盘

### 5.4 第四优先：低年级阅读负担控制

目标：

- 让低年级玩家学得进去
- 不是把内容做得过于幼稚或完全无挑战

## 6. 第一阶段推荐修改方向（普通流程）

下面是这次重写后，普通流程里真正建议先做的 5 类内容。

## 6.1 统一“普通学习事件”定义

不再只把“答对题目”当成学习事件，建议统一纳入：

- 收到新词
- 完成轻交互判断
- 完成普通挑战
- 通过 Word Gate
- 完成村庄训练
- 完成一次复盘

这样后续奖励、收藏、图鉴、成长都能复用同一套入口。

## 6.2 把收词升级为“收词 + 场景语义强化”

每次收词后，不一定都弹更长内容，但应逐步支持：

- 当前群系相关提示
- 当前敌人 / 材料相关提示
- 主题归属
- 图鉴 / 收藏进度提示

## 6.3 把普通奖励从“分数驱动”升级为“成长驱动”

普通流程学习后可见收益建议优先落到：

- 宝箱升级
- 收藏进度
- 图鉴页解锁
- 村庄训练材料
- 后续门前准备收益

## 6.4 把村庄升级为普通流程学习中枢

第一阶段可以先从这些方向起步：

- 主题词训练
- 轻量复盘
- 学习奖励领取点
- 收藏 / 图鉴查看点

## 6.5 把 Word Gate 与复活学习从“阻挡惩罚”调成“理解与恢复”

目标不是取消它们，而是调性升级：

- Word Gate 更像“理解门槛”，而不是硬拦路
- 复活学习更像“恢复机会”，而不是额外惩罚

## 7. 面向不同阶段的普通流程适配

## 7.1 幼儿园

普通流程重点应是：

- 听音认物
- 图片与物体联想
- 最少阅读负担

不建议直接作为当前主版本展开。

## 7.2 小学低年级（当前主版本）

普通流程重点应是：

- 基础词汇暴露
- 简单语义理解
- 明确的正反馈
- 村庄低压训练
- 收词与收藏成就感

## 7.3 初中

普通流程重点应是：

- 更复杂语境
- 更高主题密度
- 更强图鉴 / 收藏 / 任务系统承接

不建议在当前基础层第一阶段同时展开。

## 10. 本文档最终结论

这次校准后的普通流程结论可以浓缩成 6 句话：

1. 普通流程学习优化需要全面重写，不应只在旧计划上补几句说明。
2. 普通流程学习不能再主要依赖“多弹题”，而应更多依赖“收集、语义理解、训练、复盘、收藏”。
3. 低压层适合显性学习，中压层适合轻量学习，高压层不在本文展开。
4. 村庄应升级为普通流程学习中枢，而不是单一挑战点。
5. 收词、宝箱、图鉴、收藏都应成为学习结果的承接点。
6. BOSS / 门禁战 / 龙蛋 / 骑龙学习设计应继续保持独立专题，不回塞进普通流程基础层。
