# 普通流程学习计划重写方案（基于 2026-03-09 调研校准）

## 摘要
- **结论**：要改，而且建议**全面重写原 2026-03-06 的 4 份普通流程学习文档**，不是只补几句说明。
- **原因**：旧计划仍然正确地抓住了“奖励闭环、低年级优先、降低挫败”，但它对普通流程的学习升级仍偏向“挑战/答题/奖励”，没有按调研结论明确区分**低压/中压/高压场景**，也没有把**显性学习 + 隐性学习 + 场景理解 + 长期成长**整合成统一模型。
- **策略**：保留原文件路径，**重写内容与标题**为 2026-03-09 校准版；BOSS/门禁战与龙蛋/骑龙继续保持独立文档，不再塞回普通流程主计划。

## 关键改动
- **重写 `2026-03-06-游戏性学习性趣味性优化分析与第一阶段建议.md`**
  - 把核心框架改成：**普通流程学习 = 低压层（村庄/图鉴/复盘）+ 中压层（跑图/收集/轻交互）+ 高压边界说明（BOSS/骑龙不在本文展开）**。
  - 把“场景记忆强化”重写为：**通过敌人/道具/群系/任务语义理解来学**，不是单纯增加词卡或弹题。
  - 增加“普通流程中哪些地方适合显性练习，哪些地方要做隐性学习”的明确分配。
  - 明确普通流程后续重点：**村庄训练、收集循环、图鉴收藏、战后复盘、轻量门前提示**。

- **重写 `2026-03-06-第一阶段奖励闭环实施计划.md`**
  - 把“奖励来源”从主要围绕**挑战答题**，改为覆盖四类普通流程学习事件：**收词、轻交互判断、村庄训练、复盘回顾**。
  - 把 Task 结构改成更符合新模型的模块：
    - 学习事件统一抽象层
    - 普通流程即时反馈层
    - 宝箱/掉落/收藏绑定层
    - 村庄训练与复盘层
    - 小学低年级阅读负担与弹窗频率调优
  - 明确不做的事：**不在本计划中引入 BOSS 战内显性测验，不在本计划中展开龙成长系统**。

- **重写 `2026-03-06-第一阶段奖励闭环验收清单.md`**
  - 验收重点从“答题后是否给奖励”扩展为“普通流程中学习是否自然发生且不打断”。
  - 新增验收维度：
    - 收词是否有上下文强化
    - 村庄是否成为低压训练场
    - 宝箱/收藏/图鉴是否能承接学习结果
    - 轻交互是否帮助理解词汇而不是增加打断
    - 弹窗频率是否受控
  - 保留 BOSS/骑龙回归项，但仅作为**兼容性检查**，不作为本计划实现目标。

- **重写 `2026-03-06-第一阶段奖励闭环任务拆分-steps.md`**
  - 用新的普通流程模型重排步骤，不再以“挑战题结算”作为绝对中心。
  - steps 顺序建议改为：
    1. 统一普通流程学习事件模型
    2. 收词与场景语义强化
    3. 普通流程即时奖励与可见反馈
    4. 宝箱/图鉴/收藏绑定
    5. 村庄训练与战后复盘
    6. 小学低年级调优
    7. 回归与验收
  - 每一步都明确：**哪些是显性学习、哪些是隐性学习、哪些不能增加额外弹窗**。
