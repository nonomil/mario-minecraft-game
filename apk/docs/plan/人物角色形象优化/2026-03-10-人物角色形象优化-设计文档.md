# 人物角色形象优化设计文档

日期：2026-03-10
范围：仅视觉表现（渲染/样式），不改数值或手感

## 目标
- 主角 Steve 形象贴近经典皮肤特征，增强可读性
- 弓、箭、剑、斧、镐保持原版轮廓与识别度
- 僵尸、苦力怕、骷髅、蜘蛛更接近原版配色与关键特征

## 非目标
- 不修改伤害、冷却、速度、碰撞、掉落等数值
- 不新增资源文件或引入外部贴图
- 不改变关卡与敌人行为逻辑

## 视觉规格
### Steve（Classic，宽臂）
- 关键特征：短深棕发、棕色皮肤、深棕鼻子与胡子、白眼蓝紫瞳、青色上衣、蓝色裤子、灰色鞋
- 保留左侧衣摆下垂的非对称细节，用 1-2 像素块强化识别
- 分层策略：头发与脸部分离，衣服与裤子使用 2-3 档明暗块

### 武器（剑/斧/镐/弓/箭）
- 剑：细长直刃 + 短护手，使用高亮刃线增强锐利感
- 斧：楔形斧头 + 木柄，斧刃有明显斜切轮廓
- 镐：双齿镐头 + 短柄，齿部使用高亮块突出材质
- 弓：单片弧形轮廓，弓弦可用 1 像素线条表示
- 箭：直杆 + 金属箭头 + 尾翼，保持方向感与速度感

### 敌对生物
- 僵尸：服装与 Steve 保持一致，皮肤改为绿色并增加暗部块
- 苦力怕：亮绿色为主色，加入叶片式斑驳纹理并突出经典脸型
- 骷髅：骨白主体 + 深色眼眶，增加肋骨分块与关节细节
- 蜘蛛：深色躯干 + 高对比红色发光眼，腿部加深外缘

## 实现落点
- src/modules/14-renderer-entities.js
- drawSteve(...)：主角像素分块与持武器形态
- drawProjectile(...)：箭矢外观
- drawCreeperMob(...) / drawZombie(...) / drawSkeleton(...) / drawSpider(...)：敌方形象

## 数据与状态依赖
- 继续使用 s = width / 16 或 s = width / 26 的缩放逻辑
- 使用 playerWeapons.current、player.facingRight、player.isAttacking 判定武器外观
- 保持 armor 覆盖逻辑不变，仅更新底层像素块

## 兼容与回退
- 未识别的武器类型回退为剑外观
- 渲染失败时不影响碰撞与数值逻辑

## 测试检查
- 主角静止/移动/攻击时形象完整，无断层
- 切换剑/斧/镐/弓后轮廓可辨
- 箭矢方向正确，速度感清晰
- 僵尸/苦力怕/骷髅/蜘蛛辨识度明显提升

## 风险与权衡
- 像素块数量有限，细节需与可读性取舍
- 过度高对比可能影响背景层次，需控制明暗范围
