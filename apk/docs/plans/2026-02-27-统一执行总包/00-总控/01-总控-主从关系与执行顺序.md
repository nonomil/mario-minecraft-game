# 总控：主从关系与执行顺序

## 1. 主从关系

1. 运营主包：`01-运营执行`
2. 开发主包：`02-软件开发执行`
3. 两条线并行，但同一时段只允许一个版本处于“开发中”状态。

## 2. 版本先后

1. vA：必须先完成（共同起点）
2. vB：vA 通过后进入
3. vC：vB 后进入（并扩展 vA 的测试发音，不新增重复入口）
4. vD：vC 稳定后进入

## 3. 测试命名规范

1. 新增测试采用：`opt-YYYYMMDD-v<版本>-task<编号>-<主题>.spec.mjs`
2. 历史 `opt-0226-*` / `opt-0227-*` 保留不改名。

## 4. 冲突处理规则

1. 若多文档描述冲突，以“主执行文档”优先。
2. 运营指标与止损判断，以 `01-运营执行/03-数据看板与复盘模板.md` 为准。
3. 发布闸门，以 `02-软件开发执行/06-统一测试回归与发布闸门.md` 为准。
