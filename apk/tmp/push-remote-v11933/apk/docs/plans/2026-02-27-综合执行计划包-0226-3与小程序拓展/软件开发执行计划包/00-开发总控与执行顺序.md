# 开发总控与执行顺序

## 0. 文档主从与边界（先看）

1. `vA` 执行主文档：`docs/plans/2026-02-27-轻量商业路线综合包/07-vA-运营前修正与最小开发-逐步执行计划.md`
2. `vA` 逐小时执行文档：`软件开发执行计划包/06-vA-逐小时执行清单-4到6小时.md`（仅用于排班与打勾）
3. `vA` 其它文档（`02-vA-*`、`01-vA-P0-*`）为说明与复盘参考，不作为执行主准绳
4. 两包关系：`轻量商业路线综合包`管运营动作，`软件开发执行计划包`管代码动作；两者并行执行，`vA` 为共同起点

## 一、执行顺序（必须）

1. vA：先修最影响售后与交付的问题
2. vB：再做学习反馈增强
3. vC：再做 Web TTS 与 PWA 基线
4. vD：最后做小程序迁移底座

## 二、每个版本统一流程

1. 建分支：`feature/<version>-<topic>`
2. 写/补测试（先失败）
3. 做最小实现
4. 运行回归
5. 更新文档
6. 打版本标签（可选）

## 三、目录与关键文件

1. 页面结构：`Game.html`
2. 账号与统计：`src/modules/08-account.js`
3. 音频与TTS：`src/modules/03-audio.js`
4. 学习挑战：`src/modules/12-challenges.js`
5. 事件绑定：`src/modules/16-events.js`
6. 启动入口：`src/modules/17-bootstrap.js`
7. E2E配置：`tests/e2e/playwright.config.mjs`

## 四、统一命令清单

开发预检查：
```bash
npm -v
node -v
npm run -s
```

E2E示例：
```bash
npx playwright test -c tests/e2e/playwright.config.mjs tests/e2e/specs/opt-0226-task1-save-transfer.spec.mjs
```

## 五、统一完成定义（DoD）

1. 功能可见（页面可操作）
2. 异常可见（失败有提示）
3. 关键测试通过
4. 文档更新
5. 无主流程回归

## 六、测试命名规范（统一）

1. 新增测试统一采用：`opt-YYYYMMDD-v<版本>-task<编号>-<主题>.spec.mjs`
2. 日期统一8位：`YYYYMMDD`（示例：`opt-20260227-vA-task1-speech-test.spec.mjs`）
3. 历史 `opt-0226-*` / `opt-0227-*` 文件保留不改名，避免破坏既有引用
4. 从当前版本开始，新增文件必须使用新规范
