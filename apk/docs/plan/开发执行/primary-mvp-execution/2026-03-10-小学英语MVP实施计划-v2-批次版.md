# 小学英语 MVP 实施计划 v2（批次版）

> 适用分支：`codex/primary-mvp-execution`
> 核心思路：按“能被玩家感知的一整组闭环”来推进，每批做完就回顾

---

## 1. 适用场景

如果希望：

- 每一批完成后都有明显成果
- 更快看到玩家可感知变化
- 每批结束都能停下来评审

那就用这份批次版。

---

## 2. 批次划分

## Batch A：学习事件基础闭环

### 目标

先把学习结果“接住”。

### 涉及模块

- `src/modules/09-vocab.js`
- `src/modules/12-challenges.js`
- `src/modules/12-village-challenges.js`
- `src/modules/10-ui.js`

### 本批只做

- [ ] 收词事件标准化
- [ ] 普通挑战事件标准化
- [ ] 村庄训练事件标准化
- [ ] 至少一个轻反馈结果

### 本批不做

- [ ] BOSS 学习
- [ ] 龙蛋成长

### 批次验收

- [ ] 三类事件进入统一流
- [ ] 玩家看到至少一个结果

---

## Batch B：门前微学习闭环

### 目标

让玩家第一次真实感觉“学习能帮助战斗”。

### 涉及模块

- `src/modules/12-challenges.js`
- `src/modules/13-game-loop.js`
- `src/modules/10-ui.js`
- `src/modules/01-config.js`

### 本批只做

- [ ] 选一个门前触发点
- [ ] 接 `1` 题微学习
- [ ] 接一个轻 Buff
- [ ] 做对 / 做错都能完整走通

### 本批不做

- [ ] 阶段目标词
- [ ] telegraph words

### 批次验收

- [ ] 进入门前流程时能稳定触发
- [ ] Buff 可感知但不夸张

---

## Batch C：龙蛋成长闭环

### 目标

让玩家第一次感觉“学习在养龙”。

### 涉及模块

- `src/modules/09-vocab.js`
- `src/modules/12-challenges.js`
- `src/modules/12-village-challenges.js`
- `src/modules/10-ui.js`
- `src/modules/15-entities-boss-dragon.js`

### 本批只做

- [ ] 学习事件推进龙蛋进度
- [ ] UI 显示成长反馈
- [ ] 至少一种轻量阶段变化

### 本批不做

- [ ] 完整龙成长树
- [ ] 完整骑龙词汇 HUD

### 批次验收

- [ ] 至少三类事件推动龙蛋
- [ ] 玩家能看见成长变化

---

## Batch D：回归与收口

### 目标

统一确认 MVP 已经可玩、可感知、可继续扩。

### 本批只做

- [ ] 普通流程回归
- [ ] BOSS 入口回归
- [ ] 龙蛋主流程回归
- [ ] 更新执行记录

---

## 3. 批次执行建议

建议顺序：

1. Batch A
2. Batch B
3. Batch C
4. Batch D

每完成一批就更新一次：

- `2026-03-10-小学英语MVP执行记录.md`

---

## 4. 什么时候用 v1，什么时候用 v2

### 用 v1

如果你更担心：

- 模块耦合
- 回归风险
- 实现顺序混乱

### 用 v2

如果你更在意：

- 玩家尽快看到变化
- 每批都能停下来验收

---

## 5. 当前推荐

本项目更建议：

> 以 `v1` 为主执行顺序，以 `v2` 作为阶段汇报与验收视角。

也就是说：

- 写代码时按串行版
- 看成果时按批次版
