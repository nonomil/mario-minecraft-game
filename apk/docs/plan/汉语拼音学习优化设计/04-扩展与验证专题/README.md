# 扩展与验证专题

> 用途：原型跑通后再进入这一类文档，用来补强产品化、验证和技术边界

---

## 1. 适合什么时候看

这个分类适合在下面几种场景优先阅读：

- 听音认物和拼音小游戏已经有原型
- 准备做更完整的家长入口
- 准备安排儿童测试
- 想确认语音识别要不要进入主流程

---

## 2. 文档清单

### 家长入口

- `2026-03-10-家长控制面板设计.md`

适合回答：

- 家长最小入口应该提供什么
- 第一阶段家长面板应该轻到什么程度

### 幼儿验证

- `2026-03-10-幼儿用户测试策略.md`

适合回答：

- 测试家庭数量
- 观察指标
- 真实用户验证要点

### 补充专题总览

- `2026-03-10-幼儿园模式补充专题规划.md`

适合回答：

- 当前还有哪些专题没补齐
- 这些专题为什么要放到后面做

### 语音识别边界

- `2026-03-10-幼儿园模式语音识别技术方案.md`

适合回答：

- 为什么第一阶段不要把语音识别做成主依赖
- 语音输出和语音输入应该怎么区分

---

## 3. 当前使用建议

这一组文档不建议和第一阶段最小闭环同时大规模展开。

当前更适合：

1. 先完成基础原型
2. 再安排验证和产品化补强
3. 最后决定是否扩展到语音识别和更完整的家长面板

---

## 4. 这一类文档的定位

这一组文档本质上是：

- 风险控制层
- 用户验证层
- 产品扩展层

它们的作用不是“先把功能做多”，而是：

> 在原型已经成立的前提下，帮助这条线进入更稳的下一阶段。
