# 幼小衔接年级分层设计

## 背景

- 当前 `汉字模式`、`拼音模式`、`幼小衔接语文` 已具备 `gradeBand` 字段，但运行时仍按整包投放。
- 用户已经确认把 `学前 / 小学一年级 / 小学二年级` 做成正式分类，并要求不再停下来确认，直接落地。
- 现有词库里同时存在精确层级（`学前` / `一年级` / `二年级`）与跨层级标签（`学前-一年级` / `学前-二年级`），因此需要同时支持“精确筛选”和“跨层兼容”。

## 目标

- 在设置面板与首次词库弹窗中加入正式的年级范围选择。
- 让 `汉字模式`、`幼小衔接模式`、桥接包挑战题库按年级范围过滤。
- 在词卡 / HUD / 词库预览中更明确显示当前学习层级。
- 保持现有词库总量与包结构不变，优先做运行时过滤，避免破坏词库构建链路。

## 方案对比

### 方案 A：直接拆分更多词包

- 优点：每个年级包最直观，切换逻辑简单。
- 缺点：会复制 manifest 与包文件，维护成本高，也容易让自动轮换和已有测试大面积重写。

### 方案 B：保留词包，运行时按 `gradeBand` 过滤

- 优点：改动集中在设置层和运行时映射层，不破坏现有词库来源，兼容汉字 / 拼音 / 幼小衔接共用一套规则。
- 缺点：需要补一层范围匹配逻辑，并同步 UI 提示避免“看不出筛选生效”。

### 方案 C：只做 UI 分类，不做实际过滤

- 优点：实现最快。
- 缺点：用户会看到“分类”但出题内容不变，不能解决真实问题。

## 结论

- 采用 **方案 B**。
- 新增设置项 `bridgeGradeScope`，默认保持广覆盖，避免老用户升级后词量突然变少。
- 范围值采用：
  - `preschool_grade2`（默认，覆盖学前到二年级）
  - `preschool`
  - `grade1`
  - `grade2`
  - `preschool_grade1`
  - `all`

## 运行时规则

- 仅在以下学习场景应用年级过滤：
  - `vocab.bridge.*` 词包
  - `汉字模式`
  - `拼音模式`
- 过滤逻辑按“范围包含”判断：
  - `学前` → `preschool`
  - `一年级` → `grade1`
  - `二年级` → `grade2`
  - `学前-一年级` → `preschool_grade1`
  - `学前-二年级` → `preschool_grade2`
- 用户选择某一层时，跨层标签只要覆盖该层即可保留。
- 若条目没有 `gradeBand`，保留原样，避免桥接数学 / 英语意外被清空。

## UI 设计

- 设置面板新增 `学习层级` 下拉框，放在 `学习语言模式` 下方。
- 首次词库弹窗同步提供 `学习层级` 选择，降低首次进入时的门槛。
- 词库预览追加一行当前层级说明，例如：
  - `当前层级：小学一年级`
  - `当前层级：学前到二年级`
- 词条 meta 文案正式化：
  - `学前启蒙`
  - `小学一年级`
  - `小学二年级`

## 测试策略

- 先补一个新的单元回归测试，覆盖：
  - 默认设置包含 `bridgeGradeScope`
  - 设置 UI / 首次弹窗存在年级选择器
  - 运行时暴露并实现年级范围标准化与过滤函数
  - 汉字 / 幼小衔接条目能按层级命中
- 再跑现有汉字 / 幼小衔接 / 拼音回归，确认旧能力不退化。
