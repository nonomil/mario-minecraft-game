## 1. 本文档目的

这份文档只做一件事：

> 把我们现在已经整理出来的三条学习融合主线，用同一张图景讲清楚，方便后续不再把不同层级的设计混在一起。

它主要回答 4 个问题：

1. 这个项目现在的学习融合到底分几层。
2. 每一层分别解决什么问题。
3. 各层之间怎么衔接。
4. 相关文档应该先读哪几份。

## 2. 三层总模型

当前项目的学习融合，建议稳定采用下面这个三层模型：

### 第一层：普通流程基础层

负责：

- 让学习在日常游玩中自然发生
- 降低普通流程中的打断感和挫败感
- 把学习结果导入奖励、收藏、图鉴与轻复盘

典型场景：

- 跑图
- 收词
- 普通挑战
- Word Gate
- 复活学习
- 村庄
- 宝箱 / 收藏 / 图鉴

核心关键词：

- 低压显性学习
- 中压轻量学习
- 语义理解强化
- 村庄中枢
- 收藏承接

### 第二层：BOSS / 门禁战高光层

负责：

- 在不破坏高压战斗节奏前提下补入学习性
- 让“读懂词”和“读懂机制”形成联系
- 让战前准备、战中提示、战后复盘构成高光闭环

典型场景：

- Biome Gate
- 正式 BOSS 房
- 阶段切换
- 技能预警
- 战后复盘

核心关键词：

- 战前微学习
- 阶段目标词
- telegraph words
- 战后关键词复盘
- 高光奖励承接

### 第三层：龙蛋 / 骑龙长期成长层

负责：

- 承接长期学习成果
- 提供陪伴感、成长感、收藏感
- 让玩家觉得“学习不只是拿分，而是在养龙”

典型场景：

- 龙蛋唤醒
- 龙成长值
- 骑龙 HUD 词汇暴露
- 龙训练场
- 龙图鉴 / 收藏

核心关键词：

- 叠加式成长层
- 龙蛋进度
- 轻解锁
- 骑龙动作词
- 长期留存

## 3. 各层边界

为了避免后续再混乱，这里明确边界：

### 3.1 普通流程基础层不负责

- BOSS 战内的高压设计
- 门禁战前微学习细节
- 龙蛋成长公式
- 骑龙高压操作段设计

### 3.2 BOSS / 门禁战高光层不负责

- 普通跑图频率调优
- 普通收词语义强化主逻辑
- 龙长期成长系统本体

### 3.3 龙成长层不负责

- 普通流程基础频率与减压
- BOSS 战内细节提示词节奏
- 重做现有骑龙核心动作玩法

一句话说：

> 普通流程层解决“日常怎么学”，BOSS 层解决“高光时怎么学”，龙成长层解决“学完之后长期成长到哪里去”。

---

## 4. 三层之间的衔接关系

推荐衔接方式如下：

### 4.1 基础层 → 高光层

- 普通流程先建立基础词汇理解
- BOSS 层再把这些词变成机制理解与战斗优势

### 4.2 高光层 → 龙成长层

- BOSS 战后关键词复盘
- 高光奖励
- 稀有词汇达成

这些结果很适合继续喂给龙成长层。

### 4.3 基础层 → 龙成长层

- 收词
- 村庄训练
- 轻复盘
- 图鉴 / 收藏主题达成

这些是龙成长层最稳定的长期来源。

### 4.4 龙成长层 → 反哺其他两层

- 龙成长后提升陪伴感和长期目标感
- 也可反过来增强普通流程与 BOSS 层的继续游玩动力

---

## 5. 当前推荐的总体推进顺序

建议整体推进顺序是：

1. **先稳住普通流程基础层**
2. **再精修 BOSS / 门禁战高光层**
3. **最后把学习成果系统化接到龙成长层**

但就你这个项目当前状态来说，文档层已经把三层都铺出来了，所以后面真正实施时更推荐：

1. 先做最小基础层闭环
2. 再做一个最小 BOSS 高光闭环
3. 然后尽快接一个最小龙成长闭环

原因是：

- 这样玩家能更早看见“短期奖励 + 高光奖励 + 长期成长”三种不同反馈
- 也更容易验证整套学习融合思路是否真的成立

## 7. 现在这套文档最重要的统一结论

如果把我们这几天的讨论压缩成几句话，就是：

1. 学习升级不等于加更多弹窗题。
2. 普通流程要优先做“更自然的学习发生”。
3. BOSS 层要优先做“读懂提示带来战斗优势”。
4. 龙成长层要优先做“学习成果进入长期养成”。
5. 三层都要避免把学习做成新的打断源。

---

## 8. 后续使用建议

后面如果进入真正实现阶段，建议每次只拿一层里的一个最小闭环来做，不要一次跨三层同时大改。

推荐节奏：

- 一次迭代只做 `1 ~ 2` 个可感知点
- 每做完一轮就回看本索引，确认有没有越界
- 所有专题都优先服从“短、轻、准、可承接”的原则
