# 幼小衔接词库设计 Plan（Step‑by‑Step）

## Summary
把现有拼音/汉字相关词库升级为一个“幼小衔接总词库”，覆盖语文/数学/英语，并以“单一总包+子模块字段”的方式统一管理。计划包含：权威来源调研 → 词库结构与字段规范 → 旧包映射与去重 → 高质量清单生成 → QA 与回归验证。

## Step‑by‑Step 计划
1. **确认现有词库结构与选包机制**  
读取当前 `vocabManifest` 与词库格式，确认 `mode` 可用值、包加载机制、旧包 ID 兼容策略，为后续映射设计提供基线。

2. **调研与权威来源清单（联网）**  
使用 web.run 采集权威来源（教育部课程标准、幼小衔接指导、教材编写说明等），整理“语文/数学/英语”各模块的高频概念清单与样例（不直接产出成品词表，仅形成清单与依据）。

3. **定义“幼小衔接”统一词库结构（Schema）**  
确定每条词条字段与取值约束，避免新增模式破坏现有逻辑。  
固定字段建议：  
- `subject`: `language|math|english`  
- `module`: 语文/数学/英语子模块（见下步模块表）  
- `word`: 主显示文本（随 `languageMode` 变化）  
- `chinese`, `pinyin`, `english`（用于多模式显示）  
- `mode`: 保持现有值范围 `english|chinese|bilingual|pinyin`  
- `examples`（1-2 条）、`difficulty`、`stage`  
- `tags`（如“高频/必会/拓展”）

4. **子模块映射表（决策完成）**  
- 语文：拼音、识字、词语、古诗、表达  
- 数学：数与运算、逻辑推理、量与统计、图形与空间、应用题专项、综合实践  
- 英语：字母、自然拼读、单词、发音  
这些模块将映射到 `module` 字段，作为后续筛选与抽题依据。

5. **去重与合并规则（设计完成）**  
- 语文相关条目以“汉字/词语”为主键，拼音作为辅字段；  
- 拼音条目不再单独成包，融合进语文模块；  
- 英语与数学条目保留独立 `subject`，但统一进入总包。  
- 旧拼音/汉字包 ID 映射到新包 ID（保持旧存档兼容）。

6. **数据规模策略（决策完成）**  
- 总包规模目标：与当前汉字/拼音主量级对齐，并为数学/英语新增基础模块量级（规划阶段只设比例与范围，落库阶段根据调研清单确定精确数量）。  
- 语文条目优先覆盖高频字与高频词组；数学覆盖基础概念与常见题干词；英语覆盖字母、基础词与拼读单元。

7. **生成策略与流程设计**  
- 用“汉字包 + 拼音字段”为真源生成语文条目；  
- 英语/数学根据调研清单生成基础条目；  
- 建立脚本或一次性流程用于合并产出总包（在实施计划中指定文件与命令）。

8. **旧包兼容与迁移策略**  
- `vocab` 选择 ID 做映射到新包；  
- 旧 `pinyin/hanzi` 仍可被解析但指向新包（避免存档失效）。  

9. **质量校验规则**  
- 每条词条必须满足：主字段不空，`mode` 合法，`module` 合法；  
- 语文条目必须有 `pinyin`；  
- 数学条目必须有题干关键词或概念短语；  
- 英语条目必须有 `english` 字段。

10. **测试与验收**  
编写/更新以下测试以确保结构与兼容性：  
- 新包条目数量与字段完整性校验  
- 旧包 ID 映射回归  
- 典型条目显示在不同 `languageMode` 下的正确性  
- 离线 bundle 内词库一致性  

## Key Changes / Interfaces
- 新增“幼小衔接总包”并提供 `subject/module` 字段。  
- 旧拼音/汉字包 ID 映射到新包（接口不破坏）。  
- `mode` 仍使用现有枚举，避免引入新模式。  

## Test Plan
- 单元测试：字段完整性、模块合法性、映射正确性、显示回退逻辑。  
- E2E：拼音/汉字/英文模式切换后，词卡与挑战弹窗显示符合字段配置。  

## Assumptions
- 采用“单一总包 + 子模块字段”结构。  
- 保留旧包 ID 映射保证存档兼容。  
- 调研深度为“权威来源 + 高频清单”，不直接产出完整词表。  
- 词条 `mode` 不新增枚举，仅复用现有 `english/chinese/bilingual/pinyin`。
