# 内容与资源规划

> 用途：当你从“玩法成立”进入“内容怎么供给、资源怎么组织”时，优先阅读这一类文档

---

## 1. 适合什么时候看

这个分类适合在下面几种场景优先阅读：

- 想估算内容工作量
- 想规划词库和资产目录
- 想做中文到英语的桥接内容
- 想控制第一阶段内容不要铺太大

---

## 2. 文档清单

### 内容量级

- `2026-03-10-双语内容量级规划.md`

适合回答：

- 第一阶段至少要准备多少汉字、拼音、图片、音频
- 哪些主题最适合作为首包

### 内容库结构

- `2026-03-10-双语教育内容库结构设计.md`

适合回答：

- 内容按什么维度组织
- `words/` 后续应该如何分层
- 图片和音频资产如何复用

### 双语桥接内容

- `2026-03-10-双语桥接路径设计.md`

适合回答：

- 中文 / 拼音如何逐步过渡到英语
- 哪些主题最适合作为桥接内容

---

## 3. 当前使用建议

这一类文档当前不建议先于核心原型开发启动大规模实施。

更合理的顺序是：

1. 先让低龄玩法原型跑通
2. 再用这组文档控制内容和资源规模
3. 最后才做桥接内容扩展

---

## 4. 这一类文档的定位

这一组文档属于“工程支撑层”和“内容排期层”。

它们不会直接替代玩法设计，但会决定：

- 资源成本
- 内容节奏
- 后续扩展是否顺畅
