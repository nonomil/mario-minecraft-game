# 总览与决策

> 用途：先理解这条线为什么存在、和主项目怎么衔接、为什么不拆成两个独立项目

---

## 1. 适合什么时候看

这个分类适合在下面几种场景优先阅读：

- 刚接触这个目录
- 想快速理解整体目标
- 想确认这条线是不是当前项目要做的内容
- 想确认技术上为什么走单项目双模式

---

## 3. 这一类文档的作用

这一组文档不直接告诉你“先改哪个模块”，但它会决定：

- 路线是否正确
- 边界是否清楚
- 后续实现是否容易失控

换句话说，它是整个目录的“方向层”和“决策层”。

---

## 4. 当前使用建议

如果你现在要进入实现，不需要反复从头看完整目录。

只要先用这组文档确认两件事：

1. 幼儿园模式是独立交互层，不是小学模式低难度版
2. 当前应该走单项目双模式，而不是拆成两个工程

确认完这两点，就可以进入 `../02-核心设计与执行/`。
