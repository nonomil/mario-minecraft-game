# BOSS战优化 Step-by-Step 计划

## 目标
- 提升可学习性与战斗节奏
- 明确阶段化与反馈
- 增加风险/收益决策

## 验收标准（通用）
- 进入战斗后 3~5 秒内能看到明确“提示/节奏”
- 任何高伤害招式都有可识别预警
- 有至少 1 个明确的“惩罚窗口”（玩家可反击）
- 调试入口可稳定复现

## BOSS：Wither

### 设计要点
- 三阶段血量门槛（70%/35%）
- 护盾期与破盾窗口
- 攻击前摇加强（视觉/音效/提示条）

### Step-by-Step
1. 现状梳理
   - 记录现有攻击方式、伤害、冷却、移动逻辑
   - 记录现有调试入口与可视化信息
2. 阶段定义
   - Phase 1（>70%）：常规火球 + 小范围震击
   - Phase 2（35%~70%）：加入短护盾 + 直线冲刺
   - Phase 3（<35%）：高频攻击但更明显前摇
3. 护盾与破盾窗口
   - 护盾持续时间与可打断条件
   - 护盾破裂后 2 秒硬直
4. 反馈与提示
   - 护盾开启/破裂提示
   - 冲刺预警（颜色/抖动/音效）
5. 调试接口
   - 增加 phase 强制切换
   - 增加护盾开关
6. 测试
   - 进入战斗阶段切换是否正确
   - 破盾窗口可稳定触发
   - 伤害节奏符合阶段设定

### 风险点
- 护盾期间过久导致战斗拖沓
- 冲刺频率过高导致不可读
