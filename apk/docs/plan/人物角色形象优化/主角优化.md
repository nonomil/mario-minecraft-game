# Steve 角色视觉升级实施计划（Classic Steve++）

## Summary

在不改动碰撞盒、尺寸计算和调用签名的前提下，把当前 14-renderer-entities.js 里的 Steve 从“功能可用的基础像素人”升级为“更像 Minecraft 官方风格、但在本项目里更清晰耐看”的版本。
这次只做 Canvas 绘制层升级，不切 sprite 资源，不改 drawSteve(x, y, facingRight, attacking) 的入参，不改 14-renderer-main.js 和 18-village.js 的调用方式。

## Implementation Changes

### 渲染结构

- 保持 drawSteve(x, y, facingRight, attacking) 签名不变，仍以 player.width / 26 作为唯一缩放基准。
- 将 Steve 绘制拆成固定顺序的内部 helper，全部留在同一文件内：
  - drawSteveHead
  - drawSteveTorso
  - drawSteveLimbs
  - drawSteveArmorOverlay
  - drawSteveShield
  - drawSteveWeapon
- 固定图层顺序为：背侧肢体 → 头部/面部 → 身体/腿 → 护甲覆盖 → 盾牌 → 武器。
- 不新增外部状态参数；允许在函数内部读取 player.velX 与 gameFrame，只给四肢加入 1px-2px 级别的轻微走动错位，静止时恢复近乎完全对称。

### 视觉规范

- 锁定一套高对比但仍接近经典 Steve 的配色，不让实现者自行挑色：
  - 皮肤：#D4A07A / 阴影 #B77754
  - 头发：#3A281B / 阴影 #24170F / 提亮 #5A4030
  - 上衣：#4EA7D9 / 阴影 #2D6F98 / 高光 #86D7F2
  - 裤子：#4C4A96 / 阴影 #2D2A63
  - 靴子：#7E858C / 阴影 #4F565E
  - 装饰高光：#D6B35A
- 头部必须做这几项，不留实现分歧：
  - 发际线更厚，顶部发块保留 6px 高，但补 1 级侧向体积，避免“头顶一条平板”。
  - 眼睛保持蓝瞳，但增加更清晰的眼白和 1 条深色眉线。
  - 鼻梁与下巴阴影更集中，减少脸部大面积同色块。
- 服装必须增加这几项层次：
  - 肩部缝线
  - 胸前斜肩带/背带
  - 袖口分层
  - 腰带分界
  - 靴口深色包边
- sunscreen 变体保留，但只换同一套结构下的浅色 token；不再使用一套完全不同的角色语言。
- 当前盾牌分支保留，并把盾牌挂到前臂位置，不能像独立悬浮物；装备护甲后盾牌仍必须完整可见。

### 兼容约束

- 不改玩家 width/height、攻击判定、命中点、相机逻辑。
- 不改 drawSteve 的调用点和返回语义。
- 不把角色绘制迁移到新文件，不引入图片资源，不引入第三方库。
- 不顺手改村庄 NPC、敌人、BOSS 的美术风格；本任务只改 Steve。

## Test Plan

- 新增 1 个 unit 回归：
  - 断言 drawSteve 签名未变。
  - 断言新 helper 存在且 drawSteve 会按顺序调用护甲/盾牌/武器分层。
  - 断言 sunscreen 分支仍存在。
- 新增 1 个 Playwright 视觉烟测：
  - 启动游戏后把玩家固定在可视区域中央。
  - 分别在朝左、朝右、装备盾牌、装备护甲、开启 sunscreen 五种状态下采集 canvas 中玩家包围盒的像素。
  - 断言玩家区域内能采到头发深色、皮肤色、上衣主色、裤子主色；有盾牌时额外采到盾牌金属色。
- 手工验收必须覆盖：
  - 主场景跑动/静止
  - 攻击状态
  - 村庄室内预览
  - 盾牌已装备
  - 护甲已装备
  - sunscreen 激活
  - 手机窄屏与桌面宽屏各看一次，确认没有因为新层次导致角色“糊成一块”或遮住武器/盾牌。

## Public Interfaces

- 无 public API 变化。
- 继续保持 drawSteve(x, y, facingRight, attacking) 兼容现有调用链。

## Assumptions

- “可以，继续”视为确认采用上一轮推荐的 Classic Steve++ 方向，而不是改成更夸张的赛博/动漫风。
- 本任务不处理现有无关的旧测试漂移，例如 p1-crafting-foundation.spec.mjs 里那条老的合成摘要断言。
- 若实现过程中需要更多视觉细分，优先在当前文件内加私有 helper，不拆模块。
