﻿﻿# 幼小衔接词库扩充（Design）

**日期**：2026-03-13  
**目标**：在现有“幼小衔接总词库”基础上补齐语文/数学/英语的模块覆盖，使每个模块达到“可教学使用”的最小完整度。

---

## 1. 现状与问题
- 当前总包条目：约 990 条。
- 语文条目量级较大，但 **词语/古诗/表达模块缺失**。
- 数学与英语条目极少，仅为骨架示例。

## 2. 扩充策略（Phase A）
**总体策略**：保持现有 schema，不引入新模式；按模块补齐“最小可用规模”。

### 2.1 目标规模（Phase A）
- 语文新增：
  - 词语：200 条
  - 表达：120 条
  - 古诗：40 条
- 数学新增：120 条（6 个模块各 20 条）
- 英语新增：160 条（4 个模块各 40 条）
- 语文“控笔/描红”以 `tags` 标注（约 80 条）

### 2.2 模块映射（不变）
- 语文：拼音 / 识字 / 词语 / 古诗 / 表达
- 数学：数与运算 / 逻辑推理 / 量与统计 / 图形与空间 / 应用题专项 / 综合实践
- 英语：字母 / 自然拼读 / 发音 / 单词

## 3. 字段规范
**语文（subject=language）**
- 必填：`word`、`chinese`、`pinyin`、`module`
- `mode` 维持现有逻辑（识字/词语/古诗/表达使用 `chinese`）
- `tags`：可选 `控笔`、`描红`、`口语` 等

**数学（subject=math）**
- 必填：`word` + (`concept` 或 `keywords`)
- `mode`: `chinese`
- `keywords` 用于题干提示词

**英语（subject=english）**
- 必填：`english`、`word`、`module`
- `mode`: `english`
- `phonics` 用于自然拼读/发音模块

## 4. 生成规则与质量约束
- 语文词语：高频生活词组，词长 2-4 字为主，拼音完整标注。
- 古诗：以诗题/名句为条目，长度 ≤ 12 字，拼音完整标注。
- 表达：生活口语表达短语，可叠加 `tags: ['口语']`。
- 数学/英语条目避免重复，保证每个模块 >= 20/40 条。

## 5. 兼容性
- 不改变旧包 ID 映射逻辑。
- 不新增新的 `mode` 枚举。

## 6. 测试与验收
- 模块覆盖性校验（每个模块最小数量）
- 字段完整性校验（必填字段不空）
- 旧包映射回归（既有测试继续通过）

---

**备注**：Phase B 可在 Phase A 基础上对接外部词库扩容，并通过人工审校合入。
