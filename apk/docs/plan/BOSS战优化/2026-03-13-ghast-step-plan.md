# BOSS战优化 Step-by-Step 计划

## 目标
- 提升可学习性与战斗节奏
- 明确阶段化与反馈
- 增加风险/收益决策

## 验收标准（通用）
- 进入战斗后 3~5 秒内能看到明确“提示/节奏”
- 任何高伤害招式都有可识别预警
- 有至少 1 个明确的“惩罚窗口”（玩家可反击）
- 调试入口可稳定复现

## BOSS：Ghast

### 设计要点
- 强化火球反弹玩法
- 明确“反弹成功”奖励窗口
- 增加区域安全点

### Step-by-Step
1. 现状梳理
   - 火球速度、频率、反弹逻辑
2. 火球反弹强化
   - 反弹成功触发硬直 1.5~2s
   - 反弹成功提示（闪光/音效）
3. 进攻节奏
   - 火球发射加前摇 0.5s
   - 连续发射上限与冷却
4. 安全点机制
   - 小范围安全区或间歇空窗
5. 调试接口
   - 反弹成功强制触发
   - 火球频率快速调整
6. 测试
   - 反弹成功能稳定触发硬直
   - 玩家能在窗口期输出

### 风险点
- 反弹窗口太短导致玩法失败
