# 盾牌低门槛平衡与自动续用设计

## Summary
把盾牌定位为“常用但不过强”的消耗型木盾：制作成本低、耐久较短、自动续用，提升容错与操作顺滑度；同时放大角色手持盾牌渲染 2 倍，增强可视性。

## Goals
- 盾牌成为“随手就能做、常用”的基础防具，不再依赖铁矿。
- 盾牌使用手感更顺滑：耐久归零时自动续用下一面。
- 手持盾牌视觉更清晰，避免“太小不显眼”。

## Non-goals
- 不引入新素材或新的 UI 面板。
- 不改动其他防具体系或敌人伤害体系。

## Balance 设计（推荐）
- **配方**：木棍 ×5（去掉铁）。
- **伤害减免**：30%（从现有 45% 下调）。
- **最大耐久**：50（现有 100 的一半）。
- **每次受击耐久消耗**：10（约 5 次受击报废）。

## 玩法体验
- 盾牌更像“消耗品护具”，用完就换下一面；减少玩家装备/卸下的摩擦。
- 低成本 + 低防御：不压制其他道具，仍需玩家走位或搭配其他消耗品。

## UI/提示
- HUD 护甲条显示为：`🛡️ 盾牌 {数量} · {耐久}/{最大耐久}`。
- 背包面板保持显示库存与耐久（已有结构即可复用）。
- 盾牌破碎时自动续用弹 Toast，若库存耗尽则提示“盾牌已破损”。

## Implementation Notes
- **渲染**：`drawSteveShield` 盾牌宽高按 2 倍放大，并调整相对手臂的偏移使其对齐。
- **逻辑**：
  - 受击后耐久耗尽：若 `inventory.shield > 0`，自动消耗 1 并恢复满耐久继续装备；否则卸下。
  - `shieldState.maxDurability` 与 UI 显示统一使用新值（50）。

## Tests
- Playwright：新增盾牌合成与续用的最小行为测试（配方仅木棍、耐久数值与续用逻辑）。
- 手动：观察角色手持盾牌大小为原来的 2 倍，HUD 显示数量 + 耐久。
