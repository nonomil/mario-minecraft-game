# 幼儿园与小学模式切换技术方案

> 日期：2026-03-10
> 适用范围：幼儿园汉字拼音模式、小学生英语模式、双语桥接模式
> 关联文档：`docs/plan/汉语拼音学习优化设计/2026-03-10-幼儿园汉字拼音学习融合设计.md`

---

## 1. 本文档目的

这份文档回答：

> 项目要同时支持幼儿园汉字拼音和小学英语，技术上应该怎么切，而不把代码拆成两套互相拖累的系统。

---

## 2. 总体原则

### 2.1 共享底层事件，不共享全部交互

应该共享：

- 学习事件抽象
- 内容加载机制
- 奖励事件
- 模式配置

不应该强行共享：

- 所有 UI 呈现
- 所有题型
- 所有高压玩法接入

### 2.2 配置驱动，而不是到处写 `if`

不建议：

- 在每个模块散落年龄判断

更建议：

- 在模式配置层定义能力档
- 各模块只读取配置

---

## 3. 推荐模式结构

建议至少定义三档：

1. `kindergarten`
2. `primary_lower`
3. `bridge_cn_en`

说明：

- `kindergarten`：语音 + 图片 + 动作优先
- `primary_lower`：英语词汇主线
- `bridge_cn_en`：中文锚点过渡到英语

---

## 4. 推荐配置结构

建议后续统一到类似结构：

```json
{
  "kindergarten": {
    "audioFirst": true,
    "imageFirst": true,
    "maxTextLength": 4,
    "bossLearning": "disabled",
    "dragonHudWords": "disabled"
  },
  "primary_lower": {
    "audioFirst": false,
    "imageFirst": "contextual",
    "maxTextLength": 12,
    "bossLearning": "pre-battle-only",
    "dragonHudWords": "enabled"
  },
  "bridge_cn_en": {
    "audioFirst": true,
    "imageFirst": true,
    "maxTextLength": 8,
    "bossLearning": "light",
    "dragonHudWords": "limited"
  }
}
```

---

## 5. 模块适配建议

### `src/modules/01-config.js`

- 负责读取模式配置
- 作为模式切换主入口

### `src/modules/09-vocab.js`

- 按模式切换内容源
- 按模式切换卡片呈现方式

### `src/modules/10-ui.js`

- 根据模式切换按钮大小、文字密度、图片密度

### `src/modules/12-challenges.js`

- 小学模式可启用轻挑战
- 幼儿园模式切到听音 / 图片交互

### `src/modules/13-game-loop.js`

- 根据模式决定是否接入高压学习节点

---

## 6. 模式切换落地建议

第一阶段建议只做：

1. 启动时模式选择
2. 模式配置加载
3. 词库路径切换
4. UI 密度切换

先不做：

- 自动复杂难度判断
- 深度自适应算法

---

## 7. 本文档结论

最稳的做法是：

> 共用底层事件和配置框架，分离内容层与交互层。

这样既不会把项目拆成两套完全独立代码，也不会为了兼容低龄模式而把小学模式做坏。
